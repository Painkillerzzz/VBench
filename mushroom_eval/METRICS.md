## VLM 指标（Tier 2）

VLM 指标通过 Gemini API 对视频进行语义理解，逐个回答 yes/no 问题，然后转换为 0/1 数值。所有 VLM 指标统一为 **1=好，0=坏**。

### vlm_Q1_limb_motion — 肢体运动

- **问题：** 蘑菇TUTU是否有明显的肢体或身体运动（走路、跳跃、挥手、弹跳、转身等）？
- **对应坏特征：** 四肢无法移动 / 身体僵硬
- **评分：** 1=有明显肢体运动，0=无运动
- **评价方式：** VLM 观看完整视频后判断角色是否存在可辨识的肢体动作

### vlm_Q2_natural_motion — 动作自然度

- **问题：** 蘑菇TUTU的动作是否流畅自然，像一个活泼的小动物？（而非僵硬如毛绒玩具或人偶）
- **对应坏特征：** 动作卡顿僵硬
- **评分：** 1=动作活泼自然，0=僵硬如玩偶
- **评价方式：** VLM 判断角色运动是否具有生物般的灵动感，而非机械/玩偶质感

### vlm_Q3_mostly_still — 静止程度

- **问题：** 蘑菇TUTU在视频大部分时间是否保持静止或冻结状态？
- **对应坏特征：** 过长时间静止不动
- **评分：** 1=不是（角色有足够运动），0=是（大部分时间静止）
- **评价方式：** VLM 判断角色在整段视频中的活动时间占比。注意此题为反向问题，VLM 回答 yes（静止）时得分为 0

### vlm_Q4_correct_subject — 运动主体正确性

- **问题：** 视频中主要在运动的对象是否是蘑菇TUTU本身？（而非背景或其他物体在运动，蘑菇TUTU却静止不动）
- **对应坏特征：** 运动主体错误
- **评分：** 1=蘑菇TUTU是运动主体，0=其他物体在动而蘑菇TUTU不动
- **评价方式：** VLM 比较角色和环境的运动量，判断生成模型是否错误地将运动分配给了非主角对象

### vlm_Q5_strange_objects — 异常物体

- **问题：** 视频中是否出现了不应该存在的奇怪、畸变或无法辨识的物体或形状？
- **对应坏特征：** 出现奇怪/未知物体
- **评分：** 1=无异常物体，0=有异常物体
- **评价方式：** VLM 检查画面中是否有生成伪影、畸变形状或语义不合理的物体。注意此题为反向问题，VLM 回答 yes（有异常）时得分为 0

### vlm_Q6_physics — 物理合理性

- **问题：** 视频中的运动是否符合基本物理规律？（没有物体不合理地悬空、错误粘连/融合）
- **对应坏特征：** 物理规律混乱
- **评分：** 1=物理合理，0=违反物理规律
- **评价方式：** VLM 判断是否存在重力异常、物体穿模、错误粘连等物理不合理现象

### vlm_Q7_size_consistency — 体型大小一致性

- **问题：** 蘑菇TUTU的体型大小在整个视频中是否保持一致？（没有突然不合理地变大变小）
- **对应坏特征：** 不合理的大小突变
- **评分：** 1=大小一致，0=有突变
- **评价方式：** VLM 跟踪角色在视频中的体型变化，判断是否发生了非物理性的尺寸突变

### vlm_Q8_appearance_consistency — 外观一致性

- **问题：** 蘑菇TUTU的外观（颜色、纹理、衣着）在视频中是否保持一致？
- **对应坏特征：** 衣服/身体的时间一致性差
- **评分：** 1=外观一致，0=外观发生不自然变化
- **评价方式：** VLM 判断角色的视觉属性是否在帧间保持稳定，是否有颜色跳变、纹理闪烁、衣着突变

### vlm_Q9_motion_continuity — 运动连贯性

- **问题：** 蘑菇TUTU的运动和位移变化是否平滑连续？（没有突然瞬移或抖动跳跃）
- **对应坏特征：** 动作/位移不连贯
- **评分：** 1=运动连贯，0=有瞬移/跳跃
- **评价方式：** VLM 判断角色的运动轨迹是否自然连贯，是否存在位置突变或速度跳变

## 信号指标（Tier 1）

Tier 1 指标通过数值计算方法评估视频，不依赖语义理解。基于 VBench 框架的模型和算法。

### D1_mean_flow — 平均光流幅值

- **含义：** 视频中所有帧对的平均运动量
- **对应坏特征：** 主体动作缓慢
- **值域：** 0 ~ 250+（典型范围 2~50，>50 为大幅运动）
- **方向：** 越大 = 运动越剧烈；极小值（<5）表示几乎静止
- **计算方法：** 使用 RAFT 光流模型计算相邻帧的稠密光流场，取光流幅值最大的前 5% 像素的均值作为该帧对的运动量，再对所有帧对取平均
- **局限：** 无法区分主体运动和背景/相机运动，需结合 vlm_Q4 判断

### D1_static_ratio — 静止帧占比

- **含义：** 光流运动量低于阈值的帧对所占比例
- **对应坏特征：** 过长时间静止不动
- **值域：** 0 ~ 1
- **方向：** 越大 = 越多帧处于静止状态；≥0.95 表示几乎全程静止
- **计算方法：** 统计 mean_flow 低于自适应阈值（6.0 × scale/256）的帧对数量，除以总帧对数
- **融合用途：** static_ratio ≥ 0.95 时触发一票否决

### D1_is_dynamic — 是否动态

- **含义：** VBench 兼容的二值判断：视频是否包含足够的运动
- **对应坏特征：** 主体动作缓慢
- **值域：** True / False
- **计算方法：** 若运动量超过阈值的帧对数量 ≥ count_num（按视频长度自适应），则为 True

### D2_motion_smoothness — 运动平滑度

- **含义：** 视频中运动的流畅程度
- **对应坏特征：** 动作卡顿 / 帧间跳变
- **值域：** 0 ~ 1（越高越平滑）
- **方向：** 越大 = 越流畅；低值表示存在卡顿或跳变
- **计算方法：** 使用 AMT（Arbitrary Motion Trajectories）帧插值模型，对每对相邻帧计算中间帧插值，将插值帧与实际中间帧做像素级 MAE 比较。MAE 越小说明运动越可预测/越平滑。最终分数 = (255 - mean_MAE) / 255
- **VLM 审核：** Pearson r = +0.584

### D3_temporal_flickering — 时序稳定性

- **含义：** 视频帧间的像素级稳定程度
- **对应坏特征：** 帧间跳变 / 衣服、身体时间一致性差
- **值域：** 0 ~ 1（越高越稳定）
- **方向：** 越大 = 帧间越稳定；低值表示存在闪烁或突变
- **计算方法：** 计算所有相邻帧对的像素级平均绝对误差（MAE），取均值。分数 = (255 - mean_MAE) / 255。纯数学计算，无需 GPU 模型
- **VLM 审核：** Pearson r = +0.828（所有 Tier1 中最高）

### D4_subject_consistency — 主体外观一致性

- **含义：** 视频中蘑菇TUTU的视觉特征在时间上的稳定程度
- **对应坏特征：** 不合理的大小突变 / 外观漂移
- **值域：** 0 ~ 1（越高越一致）
- **方向：** 越大 = 外观越一致；<0.85 表示严重漂移
- **计算方法：** 使用 DINO ViT-B16 自监督视觉模型提取每帧的全局特征向量，计算每帧与首帧的余弦相似度 + 相邻帧间的余弦相似度，取平均值。DINO 特征对物体的形状、大小、纹理敏感
- **融合用途：** D4 < 0.85 时触发一票否决
- **本批视频统计：** mean=0.935, P5=0.803, 187 条 < 0.85

### D5_flow_acceleration — 光流加速度

- **含义：** 视频中运动速度变化的最大幅度
- **对应坏特征：** 动作/位移不连贯
- **值域：** 0 ~ 250+（典型范围 0.5~30，>50 为剧烈突变）
- **方向：** 越大 = 运动越不连贯；高值表示存在突然加速/减速/方向改变
- **计算方法：** 在 D1 的 RAFT 光流基础上，计算相邻帧对光流均值的逐帧差分（加速度），取最大绝对值。高加速度意味着运动轨迹存在不自然的突变
- **VLM 审核：** Pearson r = +0.712

### D5_flow_spatial_var — 光流空间方差

- **含义：** 光流场在空间上的不一致程度
- **对应坏特征：** 物理规律混乱（物体撕裂、错误粘连）
- **值域：** 0 ~ 1000+（典型范围 2~50，>100 表示严重不一致）
- **方向：** 越大 = 画面不同区域的运动方向/速度越不一致
- **计算方法：** 对每帧的 RAFT 光流场计算像素级幅值的方差，再对所有帧取平均。高空间方差意味着画面中存在物理上不合理的运动模式（如物体部分区域向左移动而另一部分向右）

## 融合分类规则

最终 `label` 的判定逻辑：

1. **VLM 评分判定：** vlm_score ≤ 阈值（默认 P25 分位数）→ bad
   - vlm_score 是 Q1~Q9 得分的平均值
   - 具体触发的坏特征从得分为 0 的问题中提取

2. **Tier1 一票否决：** 即使 VLM 评分正常，以下情况也判 bad：
   - D4_subject_consistency < 0.85 → 外观一致性差
   - D1_static_ratio ≥ 0.95 → 主体几乎静止

3. **bad_reasons 字段：** 分号分隔的所有触发原因列表
